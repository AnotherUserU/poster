1) Introduction

Customer churn is an important issue in the telecom industry because losing customers affects revenue and retention costs. This project aims to identify customers who are likely to churn so companies can respond earlier and take proactive actions to improve retention.

6) Conclusion

Feature engineering prepared the data by converting "Total Charges" to integer, filling NaN values with 0, applying one-hot encoding, and using ordinal encoding for "Contract" and "Churn". The baseline Random Forest reached 80% accuracy, but churn recall was only 50%, so recall improvement became the main focus. SMOTE raised churn recall to 58%, while threshold > 0.35 improved recall to 78% and max_depth 15 with 500 estimators reduced the train-test gap to 5%.
